Layer Classification

Understanding whether a blockchain operates at Layer 1 (base chain) or Layer 2 (scaling solution built on top).

What is Layer 1?

Layer 1 is the main blockchain. It is responsible for Consensus, Block creation, Security, and maintaining the Native cryptocurrency. They have their own nodes, validators, and consensus algorithms.

What is Layer 2?

Layer 2 is built on top of Layer 1. Its purpose is to increase TPS (Transactions Per Second) and reduce gas fees. Layer 2 does NOT create its own independent consensus for the base chain. Instead, it bundles transactions and inherits security from Layer 1.

Algorithm Layer Breakdown

All 10 consensus algorithms in our explorer are Layer 1 mechanisms because they are designed to establish ground-truth security and block production. Here is exactly how they operate at Layer 1:

1. Proof of Work (PoW) - Layer 1

Operates as the foundational security layer (e.g., Bitcoin). It creates the base blocks through hash power. Layer 2s (like Lightning Network) sit on top of PoW to handle fast payments without touching the slow Layer 1 consensus every time.

2. Proof of Stake (PoS) - Layer 1

Secures the base chain (e.g., Ethereum) through economic collateral (staked ETH). It finalizes the blocks. Layer 2 rollups (Arbitrum, Optimism) process transactions off-chain and submit the final state to this PoS Layer 1 for permanent validation.

3. Delegated Proof of Stake (DPoS) - Layer 1

Operates as the base layer for chains like EOS and TRON. Because DPoS is already highly scalable and fast (via elected delegates), these Layer 1s rarely need Layer 2 scaling solutions to handle their transaction load.

4. Proof of Authority (PoA) - Layer 1

Used as the base layer consensus for enterprise or consortium chains (e.g., VeChain). Pre-approved validators create the base blocks. It provides the ultimate source of truth for the consortium network.

5. Proof of History (PoH) - Layer 1

Solana uses PoH alongside PoS at Layer 1 to create an extremely fast base chain. By acting as a cryptographic clock at the base level, it attempts to provide Layer 2-like speeds natively on Layer 1 without needing external scaling layers.

6. Practical Byzantine Fault Tolerance (PBFT) - Layer 1

Used by networks like Hyperledger Fabric. It provides immediate finality at the base layer. Since blocks are finalized the moment they are created, it serves as an ironclad Layer 1 for permissioned networks.

7. Proof of Space / Capacity (PoSpace) - Layer 1

Chains like Chia use this to secure the base ledger. Disk space replaces hash power to write the Layer 1 blocks. Any future scaling would require building a Layer 2 network on top of Chia's base PoSpace consensus.

8. Proof of Burn (PoB) - Layer 1

Operates at the base layer by requiring users to burn coins to buy virtual mining rigs. This virtual power is used to validate and append the ground-truth blocks of the network.

9. Proof of Elapsed Time (PoET) - Layer 1

Uses trusted hardware (Intel SGX) to randomly select block producers for the base layer of permissioned chains (like Hyperledger Sawtooth). It dictates the core consensus of the entire system.

10. Proof of Importance (PoI) - Layer 1

NEM uses PoI to secure its base chain. The algorithm calculates importance scores to decide who harvests the next Layer 1 block, completely defining the network's ground security.
