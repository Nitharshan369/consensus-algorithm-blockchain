What is Blockchain?

A decentralized, distributed, and cryptographically secured digital ledger.

A blockchain is a distributed digital ledger that records transactions securely across many computers (called nodes) instead of storing them in one central place.

Think of it as a shared notebook:

  • Everyone has a copy of the notebook.
  • Whenever someone writes a new page (block), everyone updates their copy.
  • Once a page is added, it cannot be changed without everyone noticing.

This makes blockchain transparent, secure, and tamper-resistant.

Why was Blockchain Created?

Before blockchain, digital transactions usually required a trusted third party, such as Banks, Payment processors, or Governments. Blockchain removes the need for a central authority by allowing participants to trust the system instead of trusting each other.

Properties of Blockchain

1. Decentralized

Blockchain is not controlled by a single authority. Instead, the ledger is shared across many computers (nodes), making the network more reliable and resistant to failures or censorship.

2. Transparent

All confirmed transactions are visible to every participant on the network (in public blockchains). This allows anyone to verify the transaction history, increasing trust and accountability.

3. Immutable

Once a transaction is recorded and confirmed, it cannot be easily changed or deleted. Any attempt to modify a block breaks the chain and is rejected by the network.

4. Secured

Blockchain uses cryptographic hashing, digital signatures, and consensus mechanisms to protect data, verify transactions, and prevent unauthorized changes or fraud.

Components of Blockchain

1. Node

A computer connected to the network that stores a copy of the blockchain, verifies transactions, and helps validate and propagate new blocks.

2. Ledger

The blockchain's digital record of all transactions. It is distributed across all nodes, ensuring everyone has the same copy.

3. Wallet

Software or physical device used to store public and private keys. Allows users to send, receive, and securely sign transactions.

4. Hash

A fixed-length cryptographic value (e.g., SHA-256) acting like a digital fingerprint. Ensures integrity; small changes produce completely different hashes.

5. Nonce

A "number used once" that miners repeatedly change to generate a valid block hash. Essential in Proof of Work (PoW).

6. Mining

The process of validating transactions and creating new blocks by solving complex cryptographic puzzles for rewards.

7. Consensus Protocol

Rules that allow all nodes to agree on the validity of transactions (e.g., PoW, PoS, PBFT). Ensures only legitimate blocks are added.

Core Elements of a Blockchain Transaction

1. Inputs

Source of the funds being spent, referencing previously received, unspent transaction outputs (UTXOs) or the sender's balance.

2. Outputs

Specify where cryptocurrency is sent (recipient address and amount). May include a change output returning remaining balance to the sender.

3. Digital Signature

A cryptographic proof created using the sender's private key, verifying ownership and authenticating the transaction.

4. Transaction Fee

Amount paid to compensate miners/validators for processing. Higher fees encourage faster confirmation.

ECDSA (Elliptic Curve Digital Signature Algorithm)

ECDSA is a cryptographic algorithm used in blockchain to digitally sign transactions. It proves that a transaction was created by the legitimate owner of the funds without revealing their private key.

Why is ECDSA Used?

  • Authentication – Verifies that the transaction was created by the rightful owner.
  • Integrity – Ensures the transaction has not been modified after signing.
  • Non-repudiation – The sender cannot deny creating the transaction.

Blockchain Transaction Lifecycle

  1. Create Transaction: Sender enters recipient, amount, and fee in a wallet.
  2. Sign Transaction (ECDSA): Transaction is hashed and digitally signed using the sender's private key.
  3. Broadcast to Network: The signed transaction is sent to blockchain nodes.
  4. Enter Mempool: Valid, unconfirmed transactions wait in a temporary memory pool (Mempool). Nodes verify signatures, balances, formats, and prevent double spending.
  5. Miner/Validator Selects Transaction: Miners or validators choose transactions, often prioritizing higher fees.
  6. Consensus Process: The network agrees that the new block is valid using its consensus protocol (PoW, PoS, etc.).
  7. Block Creation: Valid transactions are packaged into a new candidate block.
  8. Block Added to Blockchain: The validated block is linked to the previous block, updating everyone's ledger.
  9. Transaction Confirmed: The block is permanently added. Subsequent blocks add more confirmations.

The 6-Confirmation Rule in Blockchain

The 6-confirmation rule is a widely used guideline—especially in Bitcoin—to determine when a transaction is considered highly secure and practically irreversible.

A transaction has one confirmation when it is included in a block. Every new block added after that increases the confirmation count by one.

Why Are More Confirmations Needed?

To reverse a transaction, an attacker must create an alternative blockchain and outpace the honest network. Catching up becomes exceedingly difficult with each added block unless they control a vast majority of the network (e.g., 51% attack).

What is a Consensus Algorithm?

A consensus algorithm is a set of rules that helps all computers (nodes) in a blockchain agree on which transactions are valid and who gets to add the next block.

Simple Definition

A consensus algorithm is the method used by a blockchain network to reach agreement on a single, correct version of the blockchain.

Analogy

Imagine 10 friends deciding where to eat. Instead of everyone choosing differently, they follow one rule (like voting). The rule that helps everyone agree is similar to a consensus algorithm.

Basic Terms [Before Learning Consensus Algorithms]

These are the core concepts you should understand before comparing consensus algorithms.

1. Scalability

The ability of a blockchain to handle an increasing number of transactions without slowing down or becoming expensive.

  • Block Creation Time: The average time required to create a new block (e.g., Bitcoin ~10 mins vs Solana ~0.4s).
  • TPS (Transactions Per Second): How many transactions a blockchain can process every second.
2. Security

The ability to protect data, transactions, and the consensus process from attacks. Key threats include:

  • 51% Attack: When a single entity controls >50% of mining/staking power, allowing them to reverse transactions or double spend.
  • Sybil Attack: An attempt to gain network influence by creating many fake identities or nodes.
3. Decentralisation

How widely control and decision-making are distributed across independent participants.

A highly decentralized network has many independent nodes, no central authority, and high resistance to censorship.

4. Smart Contract Support & Layers
  • Smart Contracts: Self-executing programs on a blockchain (e.g., Ethereum uses Solidity).
  • Layer 1 (Base Blockchain): The main independent network that provides consensus and security (e.g., Bitcoin, Solana).
  • Layer 2 (Scaling Solution): A secondary network built on Layer 1 to improve speed and lower costs (e.g., Lightning Network, Arbitrum).
