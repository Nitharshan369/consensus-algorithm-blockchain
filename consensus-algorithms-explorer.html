// 01 fundamentals

What is a blockchain, and what is "consensus"?

Before comparing algorithms, it helps to be precise about the two ideas underneath all of them.

A.

Blockchain

A blockchain is a shared, append-only ledger split into blocks. Each block bundles a batch of transactions and stores a cryptographic fingerprint (hash) of the block before it. That link is what makes the chain: change one old block, and every hash after it breaks — so tampering becomes obvious. No single company owns the copy; thousands of computers ("nodes") each hold an identical replica.

B.

The problem consensus solves

If nobody is in charge, who decides which transactions are real and which block gets added next? Some nodes might be slow, offline, or actively lying. Consensus is the rulebook that lets thousands of strangers, who don't trust each other, agree on one single history — without a referee.

C.

The blockchain trilemma

Every consensus design juggles three goals that resist being maximised together: Scalability (how fast, how many transactions per second), Security (resistance to attackers rewriting history), and Decentralisation (how many independent parties actually participate). Push one up, and one of the others usually gives way.
